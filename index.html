초록 (Abstract)

Abstract

장내 마이크로바이옴은 대장암(CRC) 발병과 깊이 연관되어 있으며, 비침습적 진단 바이오마커로서의 가능성이 주목받고 있다. 본 연구는 공개 샷건 메타게노믹스 데이터(ZellerG_2014: 개발 코호트 n=290; YachidaS_2019: 외부 검증 코호트 n=212)를 활용하여 로지스틱 회귀(LR), 랜덤 포레스트(RF), XGBoost(XGB) 세 가지 머신러닝 모델을 비교 분석하였다.

조성 데이터의 특성에 맞는 CLR 변환과 Rarefaction(depth=10,000), 유병률 필터링(≥5%)으로 전처리하였으며, 부트스트랩 신뢰구간(2,000회), DeLong 검정, McNemar 검정을 통해 통계적으로 엄밀하게 모델을 비교하였다. Youden J 임계값은 훈련셋에서만 산출하여 데이터 유출을 방지하였다.

개발 코호트에서 XGB가 AUC 0.998(95% CI: 0.991–1.000)으로 최고 성능을 보였으나, 외부 검증에서 LR의 Recall이 0.020으로 극단적으로 저하되는 임계값 일반화 실패 현상이 관찰되었다. DeLong 검정에서 유의하지 않던 XGB-LR 비교(p=0.811)가 McNemar 검정에서는 고도로 유의(p<0.001)하게 나타나 AUC 단독 평가의 한계를 실증하였다.

본 연구는 마이크로바이옴 기반 암 예측 모델의 임상 적용 시 AUC 외에도 분류 패턴, 임계값 안정성, 보정 성능을 종합적으로 평가해야 함을 강조하며, 이를 위한 다중 지표 검증 프레임워크를 제시한다.

⚠ 핵심 발견: 임계값 일반화 실패 (Threshold Generalizability Failure)

LR 모델은 개발 코호트에서 최적 임계값 0.986을 사용할 때 외부 코호트에서 Recall=0.020이라는 극단적 저하를 보였다. 이는 개발 코호트 특이적 확률 분포가 외부 코호트에 일반화되지 않는 현상으로, AUC 단독 평가가 분류기의 실제 임상 성능을 과대평가할 수 있음을 시사한다.

✔ 권장 모델: XGBoost

XGB는 개발/외부 코호트 모두에서 가장 안정적인 AUC를 유지하며, 분류 패턴에서도 LR보다 유의하게 우수함(McNemar p<0.001). 단, Recall(0.353)과 보정 성능(ECE=0.200) 측면에서 추가 개선이 필요하다.

⚠ DeLong–McNemar 불일치 (Dissociation)

외부 코호트에서 DeLong 검정은 XGB와 LR의 AUC 차이를 유의하지 않다고 판정(p=0.811)하였으나, McNemar 검정은 분류 패턴에서 고도로 유의한 차이를 발견(p<0.001)하였다. 이 불일치는 AUC가 순위 기반 척도로서 개별 분류 결정의 일관성을 반영하지 못함을 보여주며, AUC 단독 비교에 의존하는 현행 평가 관행의 재검토가 필요함을 시사한다.

# ─── 6. McNemar 검정 (Yates 보정) ─────────────────
mcnemar_test <- function(y, p1, p2) {
  b <- sum(p1 != y & p2 == y)
  c_ <- sum(p1 == y & p2 != y)
  chi2 <- (abs(b - c_) - 1)^2 / (b + c_)
  pchisq(chi2, df=1, lower.tail=FALSE)
}

import numpy as np

def clr_transform(counts, pseudocount=1.0):
    counts = counts + pseudocount
    log_c = np.log(counts)
    return log_c - log_c.mean(axis=1, keepdims=True)

def prevalence_filter(counts, cutoff=0.05):
    mask = (counts > 0).mean(axis=0) >= cutoff
    return counts[:, mask]

# ─── 2. Youden 임계값 ─────────────────────────────
def youden_threshold(y_true, p_hat):
    thresholds = np.linspace(0, 1, 201)
    best_j, best_t = -1, 0.5
    for t in thresholds:
        preds = (p_hat >= t).astype(int)
        tp = ((preds == 1) & (y_true == 1)).sum()
        tn = ((preds == 0) & (y_true == 0)).sum()
        sens = tp / (y_true == 1).sum()
        spec = tn / (y_true == 0).sum()
        J = sens + spec - 1
        if J > best_j: best_j, best_t = J, t
    return best_t
